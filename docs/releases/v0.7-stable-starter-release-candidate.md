## Native `file://` Validation Limitation

The available Chromium installation is controlled by an organization policy with a global URL blocklist. The browser blocks direct navigation to all URLs, including `file://` pages, before Depth Engine code can run.

Therefore this environment cannot honestly certify:

- unrestricted direct `file://` navigation;
- native persistent storage on a file origin;
- a real browser download prompt;
- a real file-picker import interaction.

The runtime audit still exercised the same DOM, scripts, event handlers, combat, equipment, save API, import normalization, export filename selection, and reset logic inside Chromium. Storage used an in-memory `localStorage`-compatible implementation because the blocked opaque origin cannot expose native storage.

This is an environment limitation, not a passing direct-file result. The pull request should remain a draft until the following check is completed in an unrestricted browser.

## Remaining Manual Release Gate

From a clean clone or downloaded source archive:

1. Open `index.html` directly in Chrome, Edge, Firefox, or Safari.
2. Confirm no console errors.
3. Confirm Rat Cellar loads by default.
4. Load Arena Waves, Sewer Patrol, Depth Kit Lab, and Crystal Mines through the selector.
5. Reload each example and confirm its own save persists.
6. Complete at least one fight and confirm XP, currency, and stage progression update.
7. Confirm loot can enter inventory and an item can be equipped.
8. Export a save and confirm the active example filename.
9. Import that save through the browser file picker.
10. Reset and confirm a clean current-version state.

After those checks pass, this candidate can be marked ready for review and considered for merge/tagging.
